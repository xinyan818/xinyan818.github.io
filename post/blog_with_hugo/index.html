使用Hugo建立博客

2022/09/20

hugo |

Hugo是一个用Go语言编写的静态网站生成器，想了解Hugo可以前往官网。

这次使用Hugo来写博客，并发布到github pages，之前就用Jekyll搞过一次（没写起来）。

为什么要用Hugo?

放在今天已经不算新鲜，但对我来说，主要就是图一新鲜（我没整过），其实Jekyll用起来也很简单，网上的教程几乎都是手把手教学。有个趁手的工具持续输出内容才是关键。

安装Hugo

官网有介绍，我这次选择源码安装(MacOS)

mkdir $HOME/src
cd $HOME/src
git clone https://github.com/gohugoio/hugo.git
cd hugo

go install
go build -o hugo main.go

执行以上命令后在$HOME/src中会有一个hugo的可执行文件，添加到环境变量，(我的shell是zsh)在 ~/.zshrc中添加以下内容后执行source ~/.zshrc

export HUGO_HOME="$HOME/src/hugo"
export PATH="$HUGO_HOME:$PATH"

在命令行中输入hugo version验证是否安装成功

使用

创建项目

hugo new site my-site-name

创建后的hugo项目是没有主题的，但是需要主题才能使用

到这个网站查看主题，选择一个中意的进行下载，我这里选择了一个简单的主题Blonde

cd my-site-name
git init
git submodule add https://github.com/opera7133/Blonde.git themes/Blonde

按照主题页面的指引安装，配置文件主题已经提供了示例，直接copy到项目根目录，不同的主题安装略有差异，具体请查看页面上的介绍

启动Hugo服务

hugo server --buildDrafts --watch

–buildDrafts或者-D，可以查看草稿内容

-–watch或者-w 选项打开的话，将会监控到内容的改动从而自动去刷新浏览器

服务起来后，进入http://localhost:1313查看

页面是空的，因为还没有内容

添加内容

hugo new posts/test.md

打开文件顶部是以下内容

---
title: "test"
date: 2022-09-20T12:18:38+08:00
draft: true
---

注意发布的时候要把draft设为false，即标记为不是草稿

发布到Github Pages上

我创建了两个仓库，一个放hugo项目，用来写博客，可能会有不完整的草稿，就设置为了私有。另一个用来部署github pages

在hugo项目中打包，将生成的pullic项目推送到github pages项目，操作如下:

hugo
cd public
git init
git remote add origin git@github.com:xinyan818/xinyan818.github.io.git
git add -A
git commit -m 'public'
git push --set-upstream origin master

非首次执行，git init会提示已存在仓库，忽略就可以
